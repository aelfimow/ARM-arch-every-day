* p.239
  - an exception causes a change of program flow
  - Exceptions
    - Interrupts
    - Memory system aborts
    - Exceptions generated by attempting to execute an instruction that is `UNDEFINED`
    - System calls
    - Secure monitor or Hypervisor traps
    - Debug exceptions
  - `SVC`: instruction causes a Supervisor Call exception
  - `BRK`: instruction generates a Breakpoint Instruction exception
  - Note: The `BRK` instruction is supported only in the `A64` instruction set. The equivalent instruction in the `T32` and `A32` instruction sets is `BKPT`.
  - `WFI`: instruction indicates that no further execution is required until a `WFI` wakeup event occurs
  - `WFE`: instruction indicates that no further execution is required until a `WFE` wakeup event occurs
  - `YIELD`: instruction provides a hint that the task performed by a thread is of low importance so that it could yield
    - is a `NOP` hint instruction.
